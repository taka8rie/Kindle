# Kindle 导书助手

一个原生 macOS 小工具，用来解决新版 Kindle 在 macOS Finder 中不显示、无法像 U 盘那样直接拖书的问题。

当前版本的目标很明确：

- 用轻量 USB 检测确认 Kindle 已连接
- 从 Mac 选择电子书文件
- 直接复制电子书到 Kindle 的 `documents` 目录
- 通过常驻 MTP agent 连接装置、读取文件缓存，并在同一个会话里复制文件

它不是一个真正把 Kindle 挂载进 Finder 的文件系统驱动；当前版本选择了更稳的应用内 MTP 会话方案，使用一个随 app 打包的 helper 连接 `libmtp`。

## 依赖

先安装 Homebrew 依赖：

```bash
brew install libmtp
```

如果你后面想进一步做成“像磁盘一样挂载”，可以在这个项目基础上继续接 FUSE / MTPFS；但那会比当前 MVP 复杂很多。

## 运行

```bash
swift run
```

也可以直接用 Xcode 打开这个目录下的 `Package.swift`。

## 打包成 .app

如果你想生成一个可双击启动的应用：

```bash
./scripts/build_app.sh
```

生成完成后，应用会出现在：

```bash
dist/Kindle 导书助手.app
```

## 使用步骤

1. 用支持数据传输的 USB 线连接 Kindle。
2. 确认 Kindle 不是只在充电状态。
3. 打开应用，点击“检查依赖”。
4. 点击“USB 检测”。
5. 点击“选择书籍”挑选 `epub`、`pdf`、`mobi`、`azw3`、`txt` 等文件。
6. 点击“复制到 Kindle”。

如果想像 calibre 的“装置”视图一样查看 Kindle 里的文件：

1. 点击“连接装置”。
2. 等待文件缓存读取完成。
3. 双击普通文件夹进入，点击“上级”返回。
4. 点击“更新缓存”可以复用同一个 MTP 会话重新读取文件树。

注意：Kindle Paperwhite 6 在打开 MTP 文件会话时可能会从“已连接计算机”页面回到图书馆，这属于设备侧 UI 行为。当前版本通过常驻 agent 尽量避免反复打开/关闭 MTP 会话。

## 今晚完成的工作

- 修复 USB 检测卡死：外部命令改为后台执行，并为命令管道读取和超时处理做了保护。
- 将导入流程扩展为批量复制，支持选择多个文件，也支持拖拽文件到应用。
- 新增 Kindle 信息区，显示 USB 检测到的型号、可用容量和总容量。
- 新增应用内 Kindle 文件视图，可浏览装置文件缓存并区分普通文件夹、文件和 Kindle `.sdr` 资料夹。
- 参考 calibre 的 MTP 设计，将短命令式调用升级为常驻 MTP agent：打开一次会话，后续读取缓存和复制文件复用同一个连接。
- 打包脚本会编译并嵌入 `mtp-kindle-ls` helper，最终生成可双击启动的 `.app`。

## 当前限制

- 依赖 Homebrew 安装的 `libmtp` 动态库。
- “USB 检测”不会打开 MTP 会话，避免部分 Kindle 从“已连接计算机”界面退回主页。
- 默认复制到 `/documents`，并兼容尝试 `/Documents`。
- 文件浏览是应用内缓存视图，不是 Finder 卷挂载。
- 还没有做删除、重命名、封面预览、格式转换。
- 还不能把 Kindle 真正挂载成 Finder 里的卷。
